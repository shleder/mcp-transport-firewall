description: Навык для реализации гибридного кэширования (LRU + SQLite) и перехвата JSON-RPC вызовов в Model Context Protocol (MCP). Используй этот навык при разработке транспортного слоя и логики проксирования.
Реализация логики MCP Context Optimizer
1. Генерация ключей кэширования (Cache Keys)
При перехвате метода tools/call от клиента, вы должны сгенерировать уникальный криптографический хэш (SHA-256), который будет служить ключом.
Формат входной строки для хэширования: server_identifier + method_name + JSON.stringify(arguments).
Сортируйте ключи объекта arguments перед стрингификацией, чтобы избежать промахов кэша из-за разного порядка ключей.

2. Алгоритм работы гибридного кэша
Когда поступает запрос:

Проверьте L1 Cache (lru-cache в оперативной памяти). Если есть Cache Hit — немедленно верните результат клиенту и залогируйте экономию токенов.

Если в L1 пусто, проверьте L2 Cache (БД SQLite). Если есть Cache Hit — верните результат клиенту, залогируйте успех и асинхронно обновите L1 Cache этим значением.

Если это полный Cache Miss:

Проксируйте запрос на целевой MCP-сервер.

Дождитесь ответа.

Верните ответ клиенту.

Асинхронно (не блокируя ответ клиенту) сохраните результат и в L1, и в L2.

3. Обработка TTL (Time-to-Live)
Добавьте поддержку инвалидации кэша. В SQLite таблице должно быть поле expires_at. При чтении из L2 проверяйте, не истек ли срок годности записи. Если истек — удаляйте запись и классифицируйте запрос как Cache Miss.

4. Сквозной пропуск (Passthrough)
Важно: проксировать через кэш нужно только методы tools/call и resources/read. Все остальные сообщения (инициализация, пинги, нотификации) должны прозрачно передаваться от клиента к серверу и обратно без изменений.
