PROJECT: MCP Context Optimizer — MCP Application Firewall
GRANT: OpenAI Codex for OSS (API credits + Codex Security access)
TARGET: Security / Developer Infrastructure

PROBLEM:
MCP stdio traffic between AI agents and local tool servers (file system, shell,
databases) is unfiltered. A compromised or misconfigured LLM can issue tool calls
that traverse restricted paths (/etc/, .env, .ssh/) or embed prompt injection
payloads targeting the tool server's execution context.

SOLUTION:
src/middleware/firewall.ts — the first open-source MCP Application Firewall.
Implements four blocking rules as composable, pluggable functions:
  1. PATH_TRAVERSAL  — regex-blocks /etc/, .env, .ssh/, /proc/, credentials
  2. PROMPT_INJECTION — blocks "ignore previous instructions", "act as root", etc.
  3. OVERSIZED_PAYLOAD — blocks params > 256KB (OOM attack vector)
  4. DANGEROUS_TOOLS — warns on exec, eval, shell, subprocess tool names

BENCHMARK: 80% cache hit ratio, 36,000 tokens saved per 100 calls, 4x latency drop.

GRANT RATIONALE:
Codex API credits and Codex Security program access are required to integrate static
vulnerability analysis into MCP transit streams — specifically, to run Codex-powered
taint analysis on tool call arguments before they reach the target server. This extends
the firewall from pattern-matching to semantic threat detection.
