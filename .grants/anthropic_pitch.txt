PROJECT: MCP Context Optimizer
GRANT: Anthropic Claude for OSS (Claude Max 20x)
TARGET: Infrastructure / Developer Tooling

PROBLEM:
AI agents operating over MCP (Model Context Protocol) against graph databases, file
systems, and knowledge bases produce N×M redundant tool calls per session. Each
repeated call consumes identical tokens, inflating context windows and triggering
context overflow on long-running tasks. At 1M-token context scales this translates
to measurable cost and reliability degradation.

SOLUTION:
An open-source transparent stdio proxy implementing a hybrid L1 (LRU, ~0.3ms) +
L2 (SQLite, persistent) cache layer in front of any MCP-compatible tool server.
Cache keys are SHA-256(server + method + params). Selective caching via
alwaysCacheTools / neverCacheTools (in mcp-optimizer.json) prevents caching of
state-mutating tools.

BENCHMARK (100 calls, 20 unique tool invocations):
  Cache Hit Ratio : 80.0%
  Tokens Saved    : ~36,000 per 100 calls
  Latency         : 110ms → 26ms (4x reduction for cached calls)

CIRCUIT BREAKER (circuit-breaker.ts):
Prevents cascade failure propagation when a target MCP server (e.g., a graph DB
adapter) becomes unavailable. State machine: Closed → Open → Half-Open. Configurable
failure threshold, recovery timeout, and half-open probe count via mcp-optimizer.json.

GRANT RATIONALE:
Claude Max 20x API limits are required to profile the proxy under realistic agentic
workloads at 200K–1M token contexts. Current free-tier rate limits prevent sustained
load testing needed to validate L2 cache eviction strategies and circuit breaker
thresholds at production scale. All findings and benchmarks will be published as
open-source tooling for the MCP ecosystem.
